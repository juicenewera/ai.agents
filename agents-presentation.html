Tópico 01 · Continuação

A Evolução
do Desenvolvedor

O nível do profissional agora se reflete na complexidade e contexto do prompt. Não é mais sobre digitar código é sobre pensar o problema.

Nível 01
Júnior

"Gerar código de login"

Foca na solução imediata com prompts básicos, sem contexto técnico ou estratégico.
Nível 02
Pleno

"Login com JWT, refresh token e rate limiting"

Define requisitos técnicos, especifica tecnologias e boas práticas de segurança.
Nível 03
Sênior

"Auth com JWT + LGPD compliance + impacto no CAC"

Visão de produto, escalabilidade, compliance e impacto no negócio incluídos no prompt.
Nível 04
Staff

"Auth corporativa: SOC2, SAML SSO, governança multi-tenant"

Arquitetura corporativa, decisões estratégicas e conformidade regulatória como padrão.

A IA nivela o campo de jogo mas quem pensa melhor o problema, entrega resultados melhores.

Tópico 02 · Boas Práticas

Vibe Coding
em Ação

O prompt inicial funciona como um briefing completo especificando o tipo de aplicativo, a stack tecnológica e as funcionalidades principais.

01
Sempre comece pela UI
O processo ideal começa pela Interface do Usuário. Estruture páginas e fluxos antes de qualquer lógica de backend.
02
Use prints como referência
Inclua screenshots do Figma ou Excalidraw para guiar a IA visualmente. Foca no frontend primeiro.
03
Dados fictícios no início
Peça à IA para usar dados fictícios para garantir que o design fique limpo e responsivo antes de conectar backend.
04
Especifique a stack desde o início
Lovable + Supabase + TypeScript no primeiro prompt evita migrações desnecessárias depois.
Exemplo de prompt inicial
"Crie um SaaS chamado [Nome]. Stack: React + TypeScript + Supabase. Páginas: - Landing page com hero e CTA - Dashboard com sidebar fixa - Tela de onboarding em 3 etapas Design: - Use dados fictícios por enquanto - Fonte Inter, cantos arredondados - Tema escuro com accent azul NÃO conecte o backend ainda."
Regra de ouro

UI linda e funcional primeiro. Backend depois. Isso evita refatorações desnecessárias e mantém o design coerente.

Tópico 03 · Agentes de IA

Estrutura básica
de um Agente

Um agente de IA vai além de um modelo de linguagem ele integra memória, percepção e ferramentas para atingir objetivos de forma autônoma.

LLM
Motor de Raciocínio
O modelo de linguagem que processa informações e decide as próximas ações.
Mem
Memória
Contexto de curto e longo prazo. O agente "lembra" do histórico e preferências.
Per
Percepção
Leitura do ambiente: inputs do usuário, dados de APIs, estado atual do sistema.
Tool
Ferramentas / Ações
Calendar(), WebSearch(), GetProfile() o agente age no mundo real via APIs.

O componente central: Objective Driven o agente age sempre orientado a um objetivo definido, não apenas respondendo perguntas.

Tópico 03 · Arquitetura

Arquitetura
com Supervisor

Sistemas mais complexos utilizam um Supervisor que roteia as demandas do usuário para agentes especializados cada um com seu próprio conjunto de ferramentas e contexto.

01
Usuário envia uma demanda
A solicitação chega ao Supervisor, que analisa a intenção e o contexto.
02
Supervisor roteia (route)
Com base na tarefa, o Supervisor decide qual agente especializado deve executar.
03
Agentes especializados atuam
Agent 1, 2, 3 cada um com seu LLM, memória e ferramentas próprias.
04
Resposta consolidada ao usuário
O Supervisor coleta e unifica as respostas antes de retornar ao usuário.

Vantagem: Cada agente especializado tem um System Prompt menor e mais focado resultando em maior precisão e menor custo.

Tópico 03 · Ecossistema

Categorias de
Agentes de IA

Há uma vasta gama de categorias sendo criadas. Conhecer o ecossistema é o primeiro passo para escolher ou construir o agente certo.

Memória
Agentes que aprendem com interações anteriores e constroem conhecimento persistente sobre o usuário.
Educação
Tutores personalizados que se adaptam ao ritmo e estilo de aprendizagem de cada aluno.
Análise de Dados
Interpretam datasets, criam visualizações e extraem insights acionáveis automaticamente.
Programação
Escrevem, revisam e depuram código. Constroem features completas a partir de descrições em linguagem natural.
Compras
Pesquisam produtos, comparam preços e executam compras de forma autônoma conforme preferências.
Segurança
Monitoram sistemas, detectam anomalias e respondem a ameaças em tempo real.
Email / Comunicação
Gerenciam caixa de entrada, redigem respostas e priorizam comunicações importantes.
Vendas / SDR
Qualificam leads, personalizam outreach e agendam reuniões com prospects automaticamente.
Suporte ao Cliente
Resolvem tickets, escalam para humanos quando necessário e aprendem com cada interação.
